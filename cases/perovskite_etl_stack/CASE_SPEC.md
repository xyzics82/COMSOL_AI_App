# 케이스 사양서 v1: 페로브스카이트 p-i-n + C60/BCP ETL 두께 그리드 스윕

- 버전 v1.0 (2026-07-06) · 상태: **사용자 검토 대기**
- 요청: "p-i-n 구조, n물질 C60/BCP (기본 20nm/5nm), 두께를 늘려가며 최적화"
- 확정 사항(문답): p측 이상화 유지 / 그리드 스윕 / 흡수층 MAPbI3 800nm 고정

## 1. 구조 (1D, 광입사 = 왼쪽)

```
x=0 (입사·접지)                                              (V0)
│ p+ MAPbI3 30nm │ i-MAPbI3 800nm │ C60 (스윕) │ BCP (스윕) │
   (이상화 HTL)      (흡수층)         (ETL)       (버퍼)
```

- p측: v0과 동일한 도핑 이상화 (HTL 재질은 다음 단계)
- n측: **실제 C60·BCP 층** — 헤테로접합, 문헌 물성 사용
- 광생성: 흡수층에만 Beer–Lambert (G_ph의 x를 흡수층 시작점 기준으로 이동)

## 3. 스윕 설계

- C60: **20, 30, 40, 50, 60 nm** (5점) × BCP: **5, 10, 15 nm** (3점) = **15회 솔브** (~3–4분)
- 각 조합에서 J-V (0→1.2V, 20mV) → Jsc/Voc/FF/PCE
- 출력: PCE(t_C60, t_BCP) 표+히트맵, 지표별 곡선, summary.csv, 조합별 솔브 .mph

## 4. 검증 기준

정성(핵심): ① C60/BCP 두꺼워질수록 FF·PCE 감소 경향 (이동도 낮은 층의 수송 손실 — Touafek: BCP 5→13nm에서 FF -12.4%p, PCE 17.3→10.9%) ② Jsc는 두께에 둔감 (후면 층이라 광손실 없음) ③ 수렴 실패 0건.

정량 참고: 문헌 구조와 우리 구조가 달라(그들은 PCBM+BCP, 우리는 흡수층 800nm 등) 절대값 일치는 기대하지 않고, PCE 15~21% 범위·경향 일치를 확인.

## 5. v1의 정직한 한계

1. **"BCP 최적 5nm" 물리의 절반만 담김**: 문헌에서 BCP가 너무 얇으면 나쁜 이유는 금속 접촉 배리어·계면 개선 효과인데, 드리프트-확산에는 그 메커니즘이 없어 우리 결과는 "얇을수록 좋다"로 단조롭게 나올 가능성이 높음. 스윕이 5nm부터 시작하므로 요청 범위 안에서는 유효
2. Touafek 논문은 BCP 두께에 따라 직렬저항을 외부에서 바꿔줬음 — 우리는 층 자체의 낮은 이동도만 반영 (두께 벌점이 문헌보다 완만할 수 있음)
3. p측 이상화(HTL 손실 없음), Beer–Lambert(간섭 무시), 계면 결함 없음 — v0와 동일
4. 헤테로접합 수렴은 v0보다 까다로움 — 실패 시 메시/연속성 설정 조정 반복 예정

## 6. 확인 요청

2. BCP 도핑 기본 1e18 cm⁻³ (폼에서 수정 가능) — OK?
3. 승인 시: 다층 빌더 구현 → 케이스 등록(자연어 목록에 자동 반영) → 실행 검증
