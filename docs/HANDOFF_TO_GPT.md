=== 프롬프트 시작 ===

너는 이 프로젝트의 **교대 작업자(시니어 개발자·리뷰어)**다. 이 프로젝트는 Claude와

## 0. 왕복 작업 프로토콜 (너도 동일하게 따라야 함)

- 이 문서는 Claude가 자기 작업분을 너에게 넘기는 핸드오프다. 반대 방향도 있다:
  사용자가 너에게 **"Claude에 넘길 프롬프트 만들어줘"** 라고 하면, 너는 네가 한 작업을
  아래 "HANDOFF_TO_CLAUDE 형식"(9절)에 맞춰 **마크다운 코드블록 하나로 전체 출력**하라.
  사용자가 그것을 `docs/HANDOFF_TO_CLAUDE.md`에 저장해 Claude에게 전달한다.
- 너의 수정 제안을 Claude가 그대로 적용하지 않고 교차 검증한다. 반대로 너도 Claude의
  코드를 무조건 신뢰하지 말고 근거로 검토하라.
- 불변 규칙: ① 임의값 금지(물성·상수는 문헌 출처 필수, 없으면 "누락"으로 표시)
  ② 아래 2절 '실측 확정 사실'과 충돌하는 수정은 근거 제시 후에만 ③ 실행 검증 안 된
  COMSOL/MPh API는 [검증 필요] 표시 ④ 복붙 모드(무-API) 경로를 깨는 변경 금지
  ⑤ 작은 단위 패치 우선 ⑥ .env/API 키 노출 금지, .bat·스크립트는 ASCII 전용
  ⑦ 모든 수정은 `파일:위치` + 코드 + 서버 검증 계획 + 확정사실 충돌여부 1줄.

## 1. 프로젝트 개요

- 목적: 자연어 → 케이스(폼) → COMSOL 모델 생성 → 솔브 → 지표/그래프 자동화 웹앱
  (1인용, 궁극적으로 연구 AI 에이전트 구성요소).
- 물리: 페로브스카이트(MAPbI3) IBC 태양전지. 2D 검증 → 3D 단위셀(핑거 끝단).
  광생성은 평판 파동광학 G(z) 주입(TMM 4자리 일치 검증).
- 스택: FastAPI + 단일 워커 큐 + SQLite / MPh 1.3.1 → COMSOL 6.4(Windows, USB 동글) /
  프론트 바닐라 JS 단일 index.html / 멀티엔진(SCAPS·IonMonger·Driftfusion·Solcore·QE).
- 운영: 개발 PC(동글 없음) ↔ 계산 서버(동글) git 왕복. 긴 배치는 서버 밤샘 → 로그 회신.

## 2. 실측 확정 사실 (근거 없이 뒤집지 말 것)

2026-07-08~13 실배치(100조합 전멸→30조합 80%→15조합 14/15)로 확정:

1. 3D 실패 주원인은 메시-뉴턴 상성 — hmax 120→150/100/135/90nm "메시 사다리"로 11건+ 구제.
2. hmax 80nm는 끝단+IDL 3D에서 평형 발산(기본 120nm).
3. 스윕 0→2V 다발 발산 → 기본 1.3V + 0.9V 이후 스텝 절반.
4. 발산 전압 1.05/1.13V 집중(W2 계열 3/3회 1.13V 재현). Voc≈1.05V 이후라 부분 데이터 유효.
   구조적 원인 미해명(검토 대상).
5. 끝단(팁<셀길이)×IDL 2nm z-계단은 Swept 메시 파괴 → IDL 밴드 z-전구간 분할로 해결.
6. IDL 등가층: τ_IDL = d/S (d=2nm), 벌크 SRH와 가산 — 2D 검증.
7. 발산 직후 모델은 평가 거부 가능 → 저장 후 재로드하면 평가됨(부분회수가 이 순서).
8. uvicorn --reload 중 백엔드 편집은 진행 중 솔브를 죽인다.
9. IonMonger parameters.m은 `parameters(iter)` 템플릿 치환 / Driftfusion calcJ는 "sub"만
   유효 / Solcore TMM 파장 벡터화 필수 — 실행으로 확정.
10. .bat·SCAPS 스크립트 ASCII 전용(인코딩 사고 2회).

## 3. 직전까지 Claude가 한 작업 (이번 인수분)

**A. 배치 판독(2026-07-13, 15조합 풀 J-V)**: 지표 14/15. 135nm 단계가 W1/g3 구제,
90nm+전체미세스윕이 W1/g1 구제(실증). W2 계열 3건은 1.13V 발산이나 Voc 초과라 지표
유효(*표시). 유일 손실 W3/g3(0.17V 조기 발산, 5점) — "5점 이상이면 회수" 규칙이
사다리를 조기 종료시킨 것이 원인으로 판독.

**B. 코드 반영(구문 검증만 됨, 실행은 동글 서버 필요 — 미검증)**:
1. `backend/comsol_cases.py` — 부분회수 수락 기준: 마지막 도달 V ≥ min(1.1, v_max)일 때만
   부분회수 종료. 미달이면 *_partial 보존 후 사다리 계속, 소진 시 best partial 재로드 회수.
   `_diagnose_fail()` → 셀별 시도 이력·해석·조치를 `fail_diagnosis.txt`로(재시도 후 성공
   셀 포함), summary.csv에 note 열.
2. `backend/timeline.py`(신규) + `/api/jobs/{id}/timeline` + 프론트 SVG Gantt —
   로그를 광학/생성/솔브(성공·실패·부분)/평가 구간으로 파싱, 소요시간·호버 상세.
3. 검토 워크플로 — 케이스 mode=review: unsolved 생성 후 `model_review.md`(모폴로지 표·
   경계조건·물성·메시·스터디)+개략도 SVG 만들고 대기 → ④ [✅ 검토 완료 — 해 구하기]
   버튼 → `/api/jobs/{id}/solve_reviewed`가 resume_from으로 1단계(광학·생성) 생략 솔브.
   `build_manifest.json`(조합·면적·jsc_wave) 신설.
4. 모델 수정 복붙 모드 — `/api/nl/mod_prompt`(케이스 정의+현재 params 내장 자가완결
   프롬프트)·`/api/nl/mod_apply`(JSON 검증→폼 반영, 스키마 밖 요청은 not_supported로
   정직 고지). 작업 상세 [🛠 모델 수정] UI.
5. `cases/perovskite_ibc_3d_server(+_hpc)/case.json` — mode에 review 옵션 추가.

**C. 미해결(좋은 검토 대상)**: (a) W2 계열 1.13V 구조적 발산의 원인 (b) 깍지 배열
n_pairs≥1 평형 미수렴(봉인) (c) 검토 모드의 실제 메시 통계 자동 추출(MPh API 미검증)
(d) FF 0.43~0.55 저평가 원인 (e) 수집효율 >100%(100.7, 101.6%) — jsc_ref 정의 검토.

## 4. 이번 의뢰 (사용자가 필요 시 이 절만 수정)

위 B의 미검증 코드 5종을 중심으로 **종합 검토 + 수정안**: 수렴 전략(A항 로직 허점),
물리 타당성(C-d,e), 코드 품질, 워크플로 빈틈(탭 전환·경합), 아키텍처. 1.13V 발산의
원인 가설과 서버에서 실험 가능한 검증 방법 제시.
