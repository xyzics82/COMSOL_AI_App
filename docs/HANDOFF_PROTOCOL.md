# Claude ↔ GPT 왕복 작업 프로토콜 (2026-07-13 제정)

이 프로젝트는 두 AI를 번갈아 사용한다. 인수인계는 항상 파일로 한다:

| 방향 | 파일 | 누가 갱신 | 사용자가 하는 일 |
|---|---|---|---|
| Claude 작업 → GPT에 의뢰 | `docs/HANDOFF_TO_GPT.md` | Claude | 내용 복사 → GPT에 붙여넣기 (+7절 첨부) |
| GPT 작업 → Claude에 의뢰 | `docs/HANDOFF_TO_CLAUDE.md` | GPT (코드블록으로 출력 → 사용자가 파일에 덮어씀) | GPT 출력 복사 → 파일 저장 → Claude에 "GPT 핸드오프 반영해줘" |

## 트리거 문구 (양쪽 AI 공통 규칙)

- 사용자가 **"GPT에 넘길 프롬프트 만들어줘"** → Claude는 그 시점까지의 작업 상태로
  `docs/HANDOFF_TO_GPT.md`를 갱신·생성한다.
- 사용자가 **"Claude에 넘길 프롬프트 만들어줘"** → GPT는 자기가 한 작업을
  `docs/HANDOFF_TO_CLAUDE.md` 형식(해당 파일의 템플릿 참조)에 맞춰 **마크다운 코드블록
  하나**로 전체 출력한다 (GPT는 파일을 만들 수 없으므로 사용자가 붙여넣어 저장).
- 받은 쪽 AI는 상대의 수정을 **그대로 적용하지 않고** 아래 '불변 규칙'과 교차 검증 후 반영한다.

## 불변 규칙 (양쪽 모두, 어느 방향이든)

1. **임의값 금지**: 출처(저자/연도/DOI) 없는 물성·상수 제안 금지. 모르면 "누락 — 사용자 확인".
2. **실측 확정 사실 존중**: `HANDOFF_TO_GPT.md` 2절의 목록. 뒤집으려면 근거 먼저.
3. **불확실 API는 [검증 필요] 표시**: COMSOL/MPh 호출은 실행 검증 전까지 신뢰하지 않는다.
4. **복붙 모드(무-API) 경로 보존**: anthropic/openai API 의존은 항상 선택 사항.
5. **작은 단위 패치**: 대규모 리팩터링은 단계별 계획과 함께만.
6. **.env·API 키 노출 금지**, .bat/스크립트는 ASCII 전용.
7. 모든 수정 제안은 `파일:함수/위치` + 코드(diff 또는 함수 전체) + 검증 계획(동글 서버에서
   최소 실행) + 확정 사실과의 충돌 여부 1줄 확인을 포함한다.
